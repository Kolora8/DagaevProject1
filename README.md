# ЗдравМонитор

Интерактивная геоинформационная витрина заболеваемости **врождёнными аномалиями** и сопутствующими показателями у детей по субъектам Российской Федерации (2015–2023).

Данные хранятся в **PostgreSQL** и обслуживаются **Go-API**. Фронтенд — **Next.js** с настоящей картой-хороплетом на Leaflet. Все 83 региона, фильтры по году и болезни, панель региона с графиком динамики, режим сравнения, прогнозирование.

Проект основан на данных открытого проекта **GISSS.RU**.

## Запуск

### 1. База данных

```bash
# Создать схему
psql "postgres://$(whoami)@localhost:5432/postgres" -c "CREATE DATABASE zdravmonitor;"
psql "postgres://$(whoami)@localhost:5432/zdravmonitor" -f db/schema.sql

# Загрузить набор данных (83 региона, 9 лет)
pip3 install psycopg2-binary
python3 db/seed_morbidity.py
```

### 2. Go-бэкенд

```bash
cd backend
export DATABASE_URL="postgres://$(whoami)@localhost:5432/zdravmonitor?sslmode=disable"
go run .           # http://localhost:8080
```

### 3. Фронтенд

```bash
cd frontend-next
npm install
npm run dev        # http://localhost:3000
```

Фронтенд проксирует `/api/*` на `http://localhost:8080` (настроено в `next.config.js`).

### Продакшн-сборка фронтенда

```bash
cd frontend-next
npm run build && npm start
```

## API

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/dataset` | Полный набор данных (83 региона, 9 лет, 5 болезней) |
| `GET` | `/api/regions` | Список регионов с текущими показателями здоровья |
| `GET` | `/api/regions/{id}` | Регион: показатели, прогнозы, оборудование |
| `PUT` | `/api/forecasts/{id}` | Редактировать прогноз |
| `POST` | `/api/regions/{id}/devices` | Добавить оборудование |

Ответ `/api/dataset` кэшируется в памяти на 5 минут. Ответ `/api/regions` — на 30 секунд.

## База данных

```
regions              — субъекты РФ (18 строк, с координатами сетки)
health_indicators    — текущие показатели здравоохранения по периоду
forecasts            — редактируемые прогнозы
devices              — медицинское оборудование

morbidity_regions    — 83 субъекта РФ + агрегат РФ
morbidity_diseases   — 5 видов болезней
morbidity            — заболеваемость: на 100 тыс. и абс. числа
water_quality_data   — качество воды по регионам
emissions_data       — выбросы по регионам
births_data          — рождаемость по регионам
```

## Данные

Набор включает **83 региона** × **9 лет** (2015–2023) × **5 болезней**:

- врождённые аномалии
- болезни системы кровообращения
- болезни нервной системы
- деформации бедра
- ихтиоз

Дополнительные показатели: качество воды, промышленные выбросы, рождаемость. Агрегаты по РФ используются как ориентир на графиках динамики.

Источник: `mapdata.js` открытого проекта [GISSS.RU](https://gisss.ru).

## Функциональность

- **Карта-хороплет** — настоящая политическая карта России (Leaflet + GeoJSON), все 83 региона
- **Фильтры** — год (2015–2023), болезнь, качество воды, выбросы
- **Панель региона** — таблица по всем болезням, график динамики (регион vs РФ), контекстные метрики
- **Прогнозирование** — Holt-Winters, ARIMA(1,1,1), SMA(3) прямо в браузере
- **Сравнение** — режим сравнения до 6 регионов с общим графиком и таблицей
- **Рейтинг** — сортируемый список регионов по текущему показателю
- **Сводная статистика** — среднее, максимум, минимум по выборке
- **Инвертирование шкалы** — для «позитивных» показателей (% безопасной воды)
- **Админ-панель** — вход по паролю, редактирование данных через `/admin`

## Конфигурация

Бэкенд читает `DATABASE_URL` из окружения:

```bash
export DATABASE_URL="postgres://user@localhost:5432/zdravmonitor?sslmode=disable"
```

Учётные данные админки задаются в `frontend-next/.env.local`:

```
ADMIN_USER=admin
ADMIN_PASSWORD=admin123
ADMIN_TOKEN=случайная_строка
```

> Замените значения по умолчанию перед любым публичным деплоем.
