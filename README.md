# ЗдравМонитор

Интерактивная геоинформационная витрина по заболеваемости **врождёнными
аномалиями** и сопутствующим показателям у детей по субъектам Российской
Федерации (2015–2023). Настоящая карта России (Leaflet) в режиме хороплета,
фильтры по году и типу болезни, панель региона с графиком динамики и
админ-панель для редактирования данных.

Проект основан на данных и подходах открытого проекта **GISSS.RU**.

**Стек:** Next.js 14 (App Router) · React 18 · TypeScript · Leaflet · Node.js
(API-маршруты Next). В каталоге также есть отдельный «скрытый» бэкенд на
**GoLang + PostgreSQL** (см. раздел ниже).

## Запуск

```bash
cd frontend-next
npm install
npm run dev          # http://localhost:3000
```

Продакшн-сборка:

```bash
npm run build && npm start
```

Приложению **не требуется отдельный бэкенд или БД** — оно работает на
статическом наборе данных (как и исходный GISSS.RU, где `mapdata.js` тоже
статический файл).

## Карта и данные

Карта — настоящая **политическая карта России (Leaflet)**: все регионы из
GeoJSON (источник — `codeforgermany/click_that_hood/russia.geojson`).
Режим **хороплета**: регионы окрашиваются по выбранной болезни и году
(заболеваемость на 100 тыс.). Над картой — выбор **года (2015–2023)** и
**типа болезни** (5 видов: врождённые аномалии, болезни системы
кровообращения, нервной системы, деформации бедра, ихтиоз) и цветовая легенда.

При клике по региону справа открывается панель:

- таблица заболеваемости по всем болезням за выбранный год (на 100 тыс. и
  абсолютные числа);
- **график динамики** выбранной болезни: регион против среднего по РФ;
- контекст за год: рождаемость, качество воды, выбросы.

**Данные.** Набор извлечён из `mapdata.js` проекта GISSS.RU и сохранён в
`frontend-next/public/morbidity.json` (83 региона × 9 лет × 5 болезней +
рождаемость, вода, выбросы и средние по РФ).

**Геометрия.** `russia.geojson` загружается автоматически из сети; при желании
положите файл в `frontend-next/public/russia.geojson` — тогда сеть не нужна.
Имена регионов из GeoJSON сопоставляются с данными в `lib/regionMatch.ts`.
Восточная часть РФ (Чукотка) пересекает 180°, поэтому отрицательные долготы
сдвигаются на +360° — иначе кусок «уезжает» в левую часть карты.

## Админ-панель

Адрес: **http://localhost:3000/admin** — вход по логину и паролю.

После входа можно редактировать данные: выбрать регион, год и болезнь,
изменить значения «на 100 тыс.» и «абсолютное число», а также рождаемость.
Изменения сохраняются в `public/morbidity.json` (через защищённые маршруты
`/api/admin/*`) и видны на карте после обновления страницы.

Логин и пароль по умолчанию — **обязательно смените**:

```
логин:  admin
пароль: admin123
```

Свои значения задаются в файле `frontend-next/.env.local`:

```
ADMIN_USER=ваш_логин
ADMIN_PASSWORD=ваш_надёжный_пароль
ADMIN_TOKEN=любая_длинная_случайная_строка
```

Аутентификация простая (cookie-сессия) и рассчитана на локальный/учебный
запуск; для публичного продакшена нужна полноценная авторизация.

## «Скрытый» бэкенд на Go + PostgreSQL (необязательно)

В каталогах `backend/` и `db/` лежит отдельный REST API на Go с PostgreSQL.
Он был сделан под исходные требования (редактируемые прогнозы, добавление
оборудования) и **в текущей версии витриной не используется** — основное
приложение работает на статических данных. Запуск:

```bash
psql -d postgres -f db/schema.sql            # создать БД и загрузить демо-данные
cd backend
go mod tidy
export DATABASE_URL="postgres://$(whoami)@localhost:5432/zdravmonitor?sslmode=disable"
go run .                                      # http://localhost:8080
```

Эндпоинты Go-API: `GET /api/regions`, `GET /api/regions/{id}`,
`PUT /api/forecasts/{id}`, `POST /api/regions/{id}/devices`.

> Следующий шаг развития: перенести данные заболеваемости (83 региона) в
> PostgreSQL и обслуживать их через Go-API вместо статического JSON.
